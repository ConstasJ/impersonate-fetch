## MODIFIED Requirements

### Requirement: Pure JavaScript main package boundary
The monorepo SHALL keep the main `impersonated-fetch` npm package as a pure JavaScript/TypeScript package that consumes generated backend packages at install/runtime and does not bundle native backend binaries in the main package tarball.

#### Scenario: Main package project builds
- **WHEN** the Nx build target for the main package runs
- **THEN** it builds the TypeScript package without compiling Go code as part of the package build target

#### Scenario: Main package depends on native package artifacts
- **WHEN** release packaging runs
- **THEN** the main package declares optional platform backend package dependencies generated by the release workflow
- **AND** the main package tarball excludes `native/*.dll`, `native/*.so`, and `native/*.dylib` files

#### Scenario: Main package is packed without native binaries
- **WHEN** package validation inspects the packed `impersonated-fetch` artifact
- **THEN** it contains JavaScript/TypeScript build output and package metadata
- **AND** it does not contain `packages/impersonated-fetch/native/*` backend binaries

### Requirement: Release-time generated backend packages
The monorepo SHALL treat scoped platform backend npm packages as generated release artifacts, not hand-maintained source packages, and the main package SHALL rely on those packages for production native backend delivery.

#### Scenario: Native package generation target runs
- **WHEN** the Nx native package generation target runs after native builds complete
- **THEN** it creates scoped platform package directories from templates and compiled artifacts

#### Scenario: Generated packages are not source packages
- **WHEN** contributors inspect normal source workspace packages
- **THEN** generated platform backend packages are absent unless a release/package-generation task has produced them

#### Scenario: Main package optional dependencies match generated packages
- **WHEN** contributors inspect `packages/impersonated-fetch/package.json`
- **THEN** each supported generated backend package has a corresponding optional dependency entry
- **AND** those optional dependencies use package names from the native backend platform mapping source of truth
